# freqtrade-okx 操作指南

本地 Freqtrade + OKX 量化練習環境。**預設模擬盤（dry-run），不碰真錢。**
所有指令都已封裝進 `Makefile`，在專案根目錄執行 `make <target>` 即可。
隨時可用 `make help` 列出全部指令。

---

## 1. 環境前提

- Python 3.12（由 `uv` 管理，`.python-version` 已鎖定）
- Freqtrade 2026.5.1、CCXT 4.5.x
- TA-Lib C 函式庫（`brew install ta-lib`）
- 已安裝 FreqUI 前端（`make install-ui`，已裝好）

底層其實是 `uv run freqtrade <子指令>`，Makefile 只是把常用組合包起來。

---

## 2. 核心循環

```
寫/改策略 → 下載歷史資料 → 回測看績效 → 調參數(hyperopt) → 模擬盤實跑 → (謹慎)實盤
```

主策略檔：`user_data/strategies/DonchianBreakout.py`（順勢通道突破，1h）
對照策略：`user_data/strategies/NostalgiaForInfinityX7.py`（社群 DCA 策略，5m，常駐部署用）
設定檔：`user_data/config.json`（主 bot）、`user_data/config.nfi.json`（NFI 對照），皆含帳密、已 .gitignore 排除

---

## 3. 日常操作流程

### 3-1. 驗證設定
```bash
make config
```

### 3-2. 下載歷史 K 線（回測前必做）
```bash
make download                  # 預設 BTC/ETH/SOL，5m，30 天
make download DAYS=60 TF=15m   # 自訂天數與時間框架
```
查看已下載資料：
```bash
make list-data
```

### 3-3. 回測
```bash
make backtest                       # 用全部已下載資料
make backtest TIMERANGE=20260518-   # 指定起始日
make backtest TF=15m STRAT=MyStrategy
```
看最近一次回測結果：
```bash
make backtest-show
```

### 3-4. 自動調參（hyperopt）
讓 Freqtrade 試上百組參數，找回測最佳：
```bash
make hyperopt                          # 預設 100 回合，SharpeHyperOptLoss
make hyperopt EPOCHS=200 SPACES="buy sell"
make hyperopt-show                     # 顯示最佳結果
```

### 3-5. 獲利曲線圖
```bash
make plot-profit       # 輸出 HTML 到 user_data/plot/
```

## 4. 開啟 Web UI（FreqUI）

### 啟動模擬盤 + UI
```bash
make trade        # 或 make ui（別名）
```
這是常駐程式，會用即時 OKX 行情模擬下單。**停止：在終端機按 `Ctrl-C`。**

### 登入
瀏覽器打開 **http://127.0.0.1:8080**

| 欄位 | 值 |
|------|-----|
| 帳號 | `leon` |
| 密碼 | `aaaaa` |

> 帳密設定在 `config.json` 的 `api_server.username / password`，要改在那裡改。

UI 可看：即時持倉、損益、K 線與進出場訊號、手動強制買/賣、暫停/繼續。

### 只看回測分析（不下單）
```bash
make webserver
```
同樣連 http://127.0.0.1:8080，但不會跑交易迴圈，適合純分析回測。

### 雙 bot 常駐（主策略 + NFI 對照）
`make trade` 是前景單一 bot（本機練習用）。要長期掛在主機並同時跑 NFI 對照組，用 Docker Compose：
```bash
make up        # 背景啟動 donchian(8080) + nfi(8081)
make ps        # 看狀態
make logs      # 看即時 log
make down      # 停止（sqlite 錢包保留）
```
- DonchianBreakout → http://127.0.0.1:8080（`config.json`）
- NostalgiaForInfinityX7 → http://127.0.0.1:8081（`config.nfi.json`）

完整部署（含 scp 機密檔、防火牆、CORS）見 [`DEPLOY.md`](DEPLOY.md)。

## 5. 改策略流程

1. 編輯 `user_data/strategies/DonchianBreakout.py`
   - `populate_indicators`：算指標（RSI、EMA…）
   - `populate_entry_trend`：進場條件
   - `populate_exit_trend`：出場條件
   - `minimal_roi` / `stoploss` / `trailing_stop`：風控
2. 重新回測比較：`make backtest`
3. 檢查有沒有偷看未來資料：`make lookahead`

開全新策略範本：
```bash
make new-strategy NAME=MyStrategy
make backtest STRAT=MyStrategy
```

## 6. 可覆寫變數一覽

不用改檔案，在指令後面帶 `KEY=value` 即可：

| 變數 | 預設 | 說明 |
|------|------|------|
| `STRAT` | `DonchianBreakout` | 策略類別名 |
| `TF` | `1h` | 時間框架 |
| `PAIRS` | `BTC/USDT ETH/USDT SOL/USDT` | 交易對 |
| `DAYS` | `30` | 下載天數 |
| `TIMERANGE` | （空） | 回測/最佳化時間範圍，如 `20260518-` |
| `LOSS` | `SharpeHyperOptLoss` | hyperopt 損失函數 |
| `EPOCHS` | `100` | hyperopt 回合數 |
| `SPACES` | `buy sell roi stoploss` | hyperopt 最佳化空間 |

## 8. 切換到實盤前（謹慎！）

1. 在 OKX 建立 API key，**只開「交易」權限、關閉「提現」**，並綁定 IP 白名單。
2. 填進 `config.json` 的 `exchange.key / secret / password`(passphrase)。
3. 把 `config.json` 的 `"dry_run"` 改為 `false`。
4. 先用**極小額** `stake_amount` 測試。
5. **把 UI 密碼 `aaaaa` 換成強密碼**，並重設 `jwt_secret_key`。

> ⚠️ 任何策略上實盤前都應充分回測 + 長時間 dry-run 驗證。
> 範例策略僅供學習，回測虧損屬正常，過往績效不代表未來。加密貨幣交易高風險，自負盈虧。
